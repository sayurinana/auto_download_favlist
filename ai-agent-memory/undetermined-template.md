# 待定项模板（精简版）

> 将每个待确认事项独立成条目，保持可单独决策。待定项确认前不得进入下一环节。

## 编写规则
1. 评分区间为 -100 ~ +100，综合考虑优势与风险。
2. 同一问题的备选方案使用相同的待定项编号（Uxxx），不同方案编号为Sxxx，评分按从低到高排序。
3. 每个方案必须说明原文位置、具体优化方案、优势与风险条目。
4. 用户确认后，仅保留采纳方案，其余方案删除并在`optimized-task.md`更新结论。
5. 文件统一存放于`ai-agent-output/YYYY-MM-DDTHH-MM-SS_UTC+8_任务简述/undetermined.md`。

## 模板
```markdown
## 🔍 待定项Uxxx - <问题描述>

### ✅ 方案Sxxx - 【建议采纳 +<分数>分】
**📍 原文位置**：（<文件路径>，第<行数>行）
> 摘录原文片段

**🎯 优化方案**：
> 具体方案描述

**⚖️ 权衡分析**：
- ✅ **优势**（+<分数>分）：……
- ❌ **风险**（-<分数>分）：……

---
```
如需列出“建议去除”方案，可将标题改为“❌ 方案Sxxx - 【建议去除 -<分数>分】”并在权衡分析中说明潜在价值。

## 示例（节选）
```markdown
## 🔍 待定项U001 - 表述冗杂优化

### ✅ 方案S001 - 【建议采纳 +70分】
**📍 原文位置**：（tasks/task-1.md，第15-17行）
> 目标描述冗长

**🎯 优化方案**：
> 用两句明确目标与交付的表述替换原段落。

**⚖️ 权衡分析**：
- ✅ **优势**（+50分）：阅读成本降低。
- ✅ **优势**（+25分）：突出关键目标。
- ❌ **风险**（-5分）：可能遗漏背景细节。

---
```
